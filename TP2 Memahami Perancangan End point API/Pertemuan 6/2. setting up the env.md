# Setting up `.env` di Laravel

## Apa itu `.env`?
`.env` adalah file konfigurasi environment yang berisi variabel sensitif dan pengaturan yang berbeda untuk tiap environment (development, testing, production). File ini tidak boleh dicommit ke repository karena mengandung data rahasia.

## Contoh struktur `.env`
```env
APP_NAME=Laravel
APP_ENV=local
APP_KEY=
APP_DEBUG=true
APP_URL=http://localhost

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=laravel_db
DB_USERNAME=root
DB_PASSWORD=

MAIL_MAILER=smtp
MAIL_HOST=smtp.mailtrap.io
MAIL_PORT=465
MAIL_USERNAME=your_email
MAIL_PASSWORD=your_password
```

## Variabel penting
- `APP_NAME`: Nama aplikasi
- `APP_ENV`: environment aplikasi (`local`, `production`, `testing`)
- `APP_DEBUG`: mode debug (`true`/`false`)
- `APP_KEY`: kunci enkripsi aplikasi (generated oleh artisan)
- `DB_*`: konfigurasi database
- `MAIL_*`: konfigurasi mail

## Cara merubah database ke mysql di .env

```php
 DB_CONNECTION=mysql
 DB_HOST=127.0.0.1
 DB_PORT=3306
 DB_DATABASE=laravel <- sesuaikan nama databasemu
 DB_USERNAME=root
 DB_PASSWORD=
```

## Langkah setup `.env`
1. Copy template: 
```bash
cp .env.example .env
```
2. Generate `APP_KEY`:
```bash
php artisan key:generate
```
3. Edit nilai di `.env` sesuai konfigurasi lokal atau server

## Keamanan & praktik terbaik
- Jangan commit `.env` ke Git (Laravel sudah menambahkan `.env` ke `.gitignore` secara default).
- Gunakan `.env.example` sebagai template untuk developer lain (tanpa nilai sensitif).
- Simpan nilai sensitif (password, API key) berbeda untuk tiap environment.
- Untuk production, rubah variabel environment lewat konfigurasi server/hosting.
